## 4. 本轮发现的问题

### P1. 错题库中的正确句补全只存在页面临时状态

现象：

- 错题页补出 `correctedSentence` 后，页面当下可以显示正确句
- 但刷新页面或重新进入后，又会重新请求生成

根因：

- 补全结果只写到了 `ErrorBankPage` 内部的 `backfilledCorrections`
- 没有写回全局 store 的 `errorBank`
- 因此既不会被 `localStorage` 持久化，也不会进入云同步

影响：

- 重复消耗 AI 请求
- 错题页看起来像“状态没记住”
- 用户对系统可靠性会失去信心

处理结果：

- 已修复
- 新增 store 写回方法，补全成功后立即更新 `errorBank`
- 增加回归测试，避免下次再退化成页面临时状态

相关文件：

- [ErrorBankPage.tsx](/Users/yuchao/Documents/GitHub/VerbaLab/src/app/pages/ErrorBankPage.tsx)
- [useStore.ts](/Users/yuchao/Documents/GitHub/VerbaLab/src/app/store/useStore.ts)
- [useStore.integration.test.tsx](/Users/yuchao/Documents/GitHub/VerbaLab/src/app/store/useStore.integration.test.tsx)

### P2. 单词卡片瀑布流默认不是“待复习优先”

现象：

- 用户进入词卡页时默认看到“最新收录”
- 但这个页面的高频目标其实是“先完成该复习的卡片”

影响：

- 进入页面后还要多做一次切换
- 默认状态服务的是展示，不是学习任务

处理结果：

- 已修复
- 默认展示方式改为 `待复习优先`
- 其他排序方式仍保留手动切换

## 5. 本轮已完成的改进

### 5.1 错题正确句补全持久化

改动：

- 为错误库新增 `setErrorBankCorrectedSentence`
- 补全正确句后立刻写回 store

收益：

- 刷新后不再重复生成
- 本地持久化和云同步能一起生效

### 5.2 词卡默认进入复习视角

改动：

- 单词卡片瀑布流默认排序改为 `待复习优先`

收益：

- 更贴合真实复习习惯
- 减少一次无意义切换

### 5.3 回归测试补强

新增测试：

- 错题正确句写回 `errorBank` 状态的集成测试

收益：

- 这类“页面能显示，但状态没落库”的问题不再只能靠人工发现

## 6. 当前仍建议继续补强的部分

以下问题本轮未直接改动，但建议作为下一轮优化目标：

1. 缺少页面级 E2E / smoke test  
当前更多是工具函数、store 和局部组件测试，真正的跨页面关键路径还没有自动化兜底。

2. 删除类操作仍大量依赖原生 `confirm()`  
虽然功能可用，但体验比较硬，也不利于统一样式和后续自动化测试。

3. 首页学习日历与跨页面高亮跳转缺少专项测试  
这些交互属于“平时不一定注意，一坏就很影响感知”的区域，值得补几条集成测试。

4. 云同步相关链路仍应继续加回归覆盖  
尤其是“多端数据合并”“复习状态合并”“首次登录对齐”这几类高风险逻辑。

## 7. 回归建议

后续每次较大的产品改动，建议至少回归以下项目：

### 必跑

- `npm run typecheck`
- `npm run test`
- `npm run build`

### 必查产品路径

1. 实验室造句成功 -> 入语料
2. 实验室造句失败 -> 入错题
3. 错题页正确句补全 -> 刷新后仍存在
4. 词卡工坊生成 -> 保存 -> 跳转详情
5. 单词卡详情复习 -> 待复习数量变化
6. 首页学习日历 -> 点击记录跳转目标页

## 8. 本轮结论

本轮产品整体主链路是通的，自动化基线也稳定。

真正需要修的不是“大而泛的重构”，而是这些影响学习体验的状态问题和默认行为问题：

- 该保存的状态必须真的保存
- 该优先展示的任务必须真的优先展示

本轮已经修掉两个高价值问题，并把其中一个补成了自动化测试。下一轮最值得继续投入的是：补页面级 smoke/E2E 测试，把现在主要靠人工发现的问题继续往前挪。
