## 1. 产品背景与目标 (Product Background & Objectives)

### 1.1 现状分析 (Current Situation)

传统口语学习产品存在以下核心问题：

- **语料喂养依赖**: 依赖"语料喂养"，导致用户在实战中产生"路径依赖"
- **复读机效应**: 只能复读 AI 句式，无法在真实高压场景（如雅思口语）中自由表达个人想法
- **逻辑断层**: 缺乏语法逻辑的系统性训练，导致输出不稳定
- **卡壳困境**: 遇到不会的表达就卡死，缺乏平替机制

### 1.2 核心目标 (Core Objectives)

| 目标 | 描述 | 衡量指标 |
|------|------|----------|
| 🎯 构建"私有弹药库" | 通过 100% 常用动词的搭配练习，让用户建立符合个人逻辑的语料库 | Personal_Corpus 句子数量 |
| 🔧 消灭"逻辑断层" | 通过"打字校验"，先理顺语法逻辑，再进行口语输出 | Grammar_Error_Bank 错误减少率 |
| 🔄 建立"平替机制" | 在卡壳时，利用万能动词进行语义替换，而非卡死在生僻词上 | 卡壳解决率 |

### 1.3 目标用户 (Target Users)

- **主要用户**: 雅思考生（目标 6.5+ 分数）
- **次要用户**: 需要提升英语表达能力的专业人士
- **特殊场景**: 浙大学生（校园场景）、工业设计师（专业场景）

## 2. 功能模块详述 (Feature Modules)

### 🛠️ 模块一：资产区 (The Foundry - 标准件库)

#### 2.1.1 核心功能
展示核心动词及其万能搭配，用户通过"刷卡片"式浏览，初步建立搭配的认知模型。

#### 2.1.2 数据结构

```
Standard_Library
├── 动词库 (Verb_Library)
│   └── 20 个万能动词 (Get, Take, Make, Do, Have, Go, Set, Keep, Give, Put, 
│                     Come, See, Know, Think, Find, Tell, Ask, Work, Feel, Need)
│
├── 词搭库 (Collocation_Library)
│   └── 每个动词关联 10-15 个高频搭配（共 240 个搭配）
│
└── 通用场景例句 (Example_Sentences)
    └── 每个搭配提供 3 个极简例句
        ├── 日常生活 (Daily Life)
        ├── 浙大校园 (ZJU Life)
        └── 工业设计 (Industrial Design)
```

#### 2.1.3 交互逻辑

```
用户进入资产区
    ↓
浏览动词列表（按频率排序）
    ↓
选择动词 → 查看搭配列表
    ↓
选择搭配 → 查看三场景例句
    ↓
标记为"已学习" → 计入学习进度
```

#### 2.1.4 界面要求

- **卡片式设计**: 每个动词/搭配独立卡片
- **进度可视化**: 学习进度条实时显示
- **场景切换**: 三场景例句支持 Tab 切换
- **搜索功能**: 支持动词/搭配关键词搜索

---

### 🧪 模块二：实验室 (The Lab - 逻辑校验与诊断)

#### 2.2.1 核心功能
通过打字输入进行"逻辑压力测试"，确保语法逻辑正确后再进行口语输出。

#### 2.2.2 交互逻辑

```
系统随机抽取搭配 + 中文语境
    ↓
用户打字输入句子
    ↓
AI 语法检查（启发式诊断）
    ↓
    ├─ 正确 → 存入 Personal_Corpus
    └─ 错误 → 存入 Grammar_Error_Bank + 提示 (Hint)
    ↓
用户修改后重新提交
```

#### 2.2.3 反馈逻辑 (启发式诊断)

**❌ 禁止行为**:
- 直接输出 Corrected Sentence
- 直接给出正确答案

**✅ 允许行为**:
- 解释错误原因（如"主谓不一致"）
- 给出提示 (Hint)，引导用户自我修正
- 提供相关语法点链接

**示例**:
```
用户输入：i make a decision yesterday.

AI 反馈:
❌ 发现 2 个语法问题：
   1. 句子首字母应大写
   2. 时间状语"yesterday"表示过去，动词需用过去式

💡 提示：请检查动词时态和句子格式
```

#### 2.2.4 数据沉淀

| 数据库 | 存储内容 | 触发条件 |
|--------|----------|----------|
| Personal_Corpus | 用户自己造出的正确句子 | 语法检查通过 |
| Grammar_Error_Bank | 错误原句 + 诊断原因 + 对应语法点 | 语法检查未通过 |

#### 2.2.5 界面要求

- **题目展示区**: 清晰显示动词搭配和中文语境
- **输入区**: 大文本框，支持多行输入
- **反馈区**: 错误高亮显示，提示清晰可见
- **参考区**: 显示标准例句（用户提交后才显示）

---

### 🚀 模块三：实战仓 (The Field - 即兴产出与调度)

#### 2.3.1 核心功能
高压下的口语即兴表达，及卡壳时的智能调度。

#### 2.3.2 交互逻辑

```
AI 模拟提问（TTS 播放雅思 Part 1/2 问题）
    ↓
用户口语回答（STT 转文字）
    ↓
    ├─ 顺利表达 → 继续
    └─ 卡壳 → 点击"卡壳"按钮
            ↓
        输入中文思路
            ↓
        AI 智能调度
            ↓
        ├─ 检索 Personal_Corpus → 推荐已知语料
        └─ 无匹配 → 引导用万能动词 Paraphrase
            ↓
用户继续表达
    ↓
最终评测（发音 + 流利度）
```

#### 2.3.3 卡壳监测 (Stuck Capture)

**触发方式**:
- 用户主动点击"卡壳"按钮
- 系统检测到长时间停顿（>5 秒）

**输入内容**:
- 中文思路（如："我想表达'坚持不懈'"）

**智能调度策略**:
```
1. 语义检索 Personal_Corpus
   ├─ 找到匹配 → 推荐用户自己的句子（优先）
   └─ 无匹配 → 进入 Step 2

2. 检索 Standard_Library
   ├─ 找到万能动词搭配 → 推荐平替表达
   └─ 无匹配 → 进入 Step 3

3. 引导 Paraphrase
   └─ "试试用简单词表达相同意思，例如..."
```

#### 2.3.4 最终评测

复用 Azure 96 分口语评测链路：
- **发音准确度**: 音素级评分
- **流利度**: 停顿次数、语速分析
- **语法多样性**: 句型复杂度分析
- **词汇丰富度**: 词汇分布统计

#### 2.3.5 界面要求

- **问题展示**: 文字 + 音频同步显示
- **录音控制**: 开始/暂停/重新录制
- **卡壳按钮**: 显眼位置，一键触发
- **评测报告**: 雷达图可视化展示

## 4. 关键技术约束 (Technical Constraints)

### 4.1 LLM Prompting

**启发式教学模式**:
```
System Prompt:
你是一位启发式英语教学专家。当学生犯错时：
1. ❌ 严禁直接输出正确句子
2. ✅ 解释错误原因（如"主谓不一致"）
3. ✅ 给出提示 (Hint)，引导学生自我修正
4. ✅ 提供相关语法点链接（可选）

示例:
学生：i make a decision yesterday.
你：❌ 发现 2 个语法问题：
       1. 句子首字母应大写
       2. 时间状语"yesterday"表示过去，动词需用过去式
    💡 提示：请检查动词时态和句子格式
```

### 4.2 STT/TTS 选型

**Azure Speech Services**:
- **STT**: 语音转文字（支持中英文混合识别）
- **TTS**: 文字转语音（雅思听力标准发音）
- **发音评测**: Azure Pronunciation Assessment（96 分标准）

**技术参数**:
```
- 采样率：16kHz
- 格式：WAV/MP3
- 延迟：<500ms
- 准确率：>95%
```

### 4.3 检索算法

**语义相似度匹配**:
```
输入：用户卡壳时的中文思路（如"坚持不懈"）
    ↓
中文 Embedding（BERT-Chinese）
    ↓
向量检索 Personal_Corpus
    ↓
    ├─ 相似度 > 0.8 → 推荐用户自己的句子
    ├─ 0.5 < 相似度 < 0.8 → 推荐万能动词搭配
    └─ 相似度 < 0.5 → 引导 Paraphrase
```

**技术栈**:
- Embedding 模型：BERT-Chinese / text2vec
- 向量数据库：Faiss / Pinecone
- 相似度算法：余弦相似度

### 4.4 性能要求

| 指标 | 要求 | 优先级 |
|------|------|--------|
| 页面加载时间 | <2 秒 | P0 |
| API 响应时间 | <500ms | P0 |
| 语法检查延迟 | <1 秒 | P0 |
| 语音评测延迟 | <3 秒 | P1 |
| 检索推荐延迟 | <1 秒 | P1 |

### 9.3 语法错误分类

| 错误类型 | 描述 | 示例 |
|----------|------|------|
| 主谓一致 | 主语和动词单复数不一致 | He go → He goes |
| 时态错误 | 动词时态与时间状语不匹配 | I go yesterday → I went yesterday |
| 冠词错误 | a/an/the 使用错误 | I have dog → I have a dog |
| 介词错误 | 介词使用不当 | at Monday → on Monday |
| 拼写错误 | 单词拼写错误 | becouse → because |
| 大小写错误 | 首字母未大写 | i → I |
| 标点错误 | 句末缺少标点 | Hello → Hello. |
